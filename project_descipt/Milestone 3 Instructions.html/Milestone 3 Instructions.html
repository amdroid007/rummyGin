Milestone 3 Instruction version 1.0 28 October 2014

Recommend Completion Date: Before the end of classes.

Context

In this milestone you will finish your game by developing a graphical user interface and integrating it with your Game Engine.

Objectives

To provide an opportunity to gain additional experience and skills with:

  • GUI Programming
  • GUI layouts and component hierarchies
  • The Observer design pattern
  • Use of sophisticated APIs.
  • Complete application development and deployment
  • Internationalization
  • Error recovery

Instructions

Complete your project using Swing and realizing the following requirements:

  • Create all the user interface elements required to play multiple games.
  • Create the user interface elements required to save and restore your game.
  • Log the game events and errors to a log file.
  • Implement the GUI panels as observers of the game engine.
  • Try to avoid implementing your own concurrent code.
  • Implement the application within a single JFrame: the only additional windows should be modal dialog boxes.
  • Externalize all your strings.
  • Package your application as a self-executing jar file.

Important Notes and Tips:

  • Make sure to get a basic (and possibly ugly) version of the application working before investing in beautification.
  • You can realize the learning objectives of this milestone with other GUI frameworks, like SWT or JavaFX. However, the final exam will involve Swing programming questions, so it's a good idea to practice this as well.
  • This milestone will require you to learn how to do specific programming tasks on your own, depending on how you want your GUI to look. However, remember that a lot of common UI programming tasks can be searched for in TaskNav.
